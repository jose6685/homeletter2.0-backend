AI提示詞說明書：屬靈抽卡片生成系統
1. 介紹
這份說明書是為你的「屬靈抽卡片」AI系統設計的完整指南。系統基於AI（如GPT-4o mini）生成個人化屬靈信件，讓用戶選主題後一鍵抽卡，得到天父般的溫暖訊息。核心是模組化prompt模板，注入隨機元素，生成流暢的信件：開頭稱呼問安、主要內容圍繞主題+三方向+兩經文、結尾兩個行動呼籲。語氣全程貫穿，確保像家書般親切。
系統特色：

多樣性：10主題 × C(17,3)三方向 × 13稱呼 × 10語氣 × 14行動選2 = 數百萬變體。
屬靈深度：融入聖經經文（和合本），強調安慰、盼望、行動。
易整合：適合App（如Flutter）或Web，後端用Python/Node.js呼叫OpenAI API。
輸出：JSON格式，便於UI顯示（e.g., 完整信件為主視圖，其他為側邊註解）。

適用場景：每日靈修、壓力釋放、教會分享。生成時間<5秒，成本低（GPT-4o-mini ~0.01 USD/張）。

3. Prompt模板（完整版）
複製以下模板到你的AI API呼叫。使用{{變數}}或f-string替換（見程式碼範例）。經文由AI生成，無需預設。
text你是一位慈愛的天父，正在親筆寫一封溫暖的屬靈信件給{{用戶暱稱或"親愛的孩子"}}。這封信圍繞{{主題}}的脈絡，語氣全程以{{信件語氣}}貫穿，從開頭到結尾一致（如溫柔鼓勵就全程溫柔）。信件長度150-250字，像家書般流暢，充滿聖經智慧、安慰與盼望。

結構（必須連貫寫成一封完整信，不要分段標號）：
1. 開頭：用{{稱呼}}溫柔問安，引入主題（例如，「孩子，願我的平安充滿你心。我看見你在{{主題}}中的掙扎...」）。
2. 主要內容：以{{三個方向}}為主軸（例如，安慰你的現在、帶來盼望的未來、堅定信心的根基），圍繞{{主題}}展開屬靈關懷、勸勉與教導。自然融入基督教真理，讓讀者感覺被聽見、被愛。隨機選擇兩節適合的聖經經文（新約或舊約皆可，和合本），直接引用並標註書名、章節:節數（例如，「如腓立比書4:7所說：神所賜出人意外的平安...」），讓經文如亮光照亮三個方向。

4. 生成邏輯與程式碼範例（Python，易整合）
在App後端或腳本中，用random模組選元素，注入prompt，呼叫OpenAI。以下是完整函數。
pythonimport random
import openai  # 需要pip install openai

# 元素陣列
themes = ["隨機", "工作 / 職場（壓力、升遷、與同事相處）", ...]  # 全11
directions = ["安慰", "盼望", ...]  # 全20
calls = ["孩子", "親愛的", ...]  # 全13
tones = ["溫柔鼓勵", ...]  # 全10
actions = ["禱告", "感恩", ...]  # 全15

def generate_card(user_theme, user_nickname="親愛的孩子"):
    # 主題處理
    if user_theme == "隨機":
        theme = random.choice(themes[1:])
    else:
        theme = user_theme
    
    # 隨機元素
    three_directions = random.sample(directions, 3)
    three_directions_str = "、".join(three_directions)
    call = random.choice(calls)
    tone = random.choice(tones)
    two_actions = random.sample(actions, 2)
    two_actions_str = "、".join(two_actions)  # 注入結尾
    
    # 注入prompt（用f-string）
    prompt = f"""你是一位慈愛的天父，正在親筆寫一封溫暖的屬靈信件給{user_nickname}。這封信圍繞{theme}的脈絡，語氣全程以{tone}貫穿，從開頭到結尾一致（如溫柔鼓勵就全程溫柔）。信件長度150-250字，像家書般流暢，充滿聖經智慧、安慰與盼望。

結構（必須連貫寫成一封完整信，不要分段標號）：
1. 開頭：用{call}溫柔問安，引入主題（例如，「{call}，願我的平安充滿你心。我看見你在{theme}中的掙扎...」）。
2. 主要內容：以{three_directions_str}為主軸（例如，安慰你的現在、帶來盼望的未來、堅定信心的根基），圍繞{theme}展開屬靈關懷、勸勉與教導。自然融入基督教真理，讓讀者感覺被聽見、被愛。隨機選擇兩節適合的聖經經文（新約或舊約皆可，和合本），直接引用並標註書名、章節:節數（例如，「如腓立比書4:7所說：神所賜出人意外的平安...」），讓經文如亮光照亮三個方向。
3. 結尾：溫柔呼籲讀者實踐{two_actions_str}（簡述如何做，每個1-2句，層次連結如「先...，然後...」），鼓勵立即行動，結束以「永遠愛你的天父」簽名。

輸出格式：JSON物件，包含：
{{
  "稱呼": "{call}",
  "完整信件": "一氣呵成的全信文字（包含開頭問安、主要內容、三方向、兩經文、三個行動呼籲與結尾）",
  "三方向": "{three_directions_str}（供顯示用）",
  "兩經文": "經文1: [引用+標註]；經文2: [引用+標註]（供顯示用）",
  "兩個行動呼籲": "{two_actions[0]}：簡短實踐解釋（1-2句）；{two_actions[1]}：簡短實踐解釋（1-2句）"
}}

確保內容正向、聖經準確、無爭議。讓這封信成為讀者的屬靈擁抱，全程{tone}！"""
    
    # 呼叫AI
    openai.api_key = 'your-api-key'
    response = openai.ChatCompletion.create(
        model="gpt-4o-mini",  # 高效低成本
        messages=[{"role": "user", "content": prompt}]
    )
    return response.choices[0].message.content  # JSON字符串，app可json.loads解析

# 使用：card_json = generate_card("壓力 / 焦慮")

5. 輸出範例與顯示建議

JSON輸出：如上快速範例，完整信件為核心。
App顯示：全屏卡片（信件文字），底部展開三方向/經文/兩個行動（可點擊分享）。加背景圖（如柔光聖經）。
測試範例：用程式碼跑3次不同主題，檢查流暢度（e.g., 兩個行動是否層次連結）。

6. 使用Tips與進階

微調：若經文不準，加prompt「優先新約經文」或「避免重複書卷」。
隱私/合規：生成匿名，無用戶數據注入。
擴展：加TTS朗讀（Google TTS），或圖像生成（DALL-E prompt："溫柔聖經卡片，主題{theme}"）。
除錯：若信太長，prompt調"400字"；若不夠屬靈，加"融入基督愛的核心"。
成本優化：用GPT-4o-mini，月1000張<10 USD。
